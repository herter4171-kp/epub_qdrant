# lean-ctx Steering

You have access to the **lean-ctx** MCP server. Its tools return the same information as native file reads and shell commands — but compressed. Use them and you fit more of the project into the same context window.

Read the tool descriptions. They tell you when each mode is appropriate. Default to lean-ctx tools over native equivalents whenever the task involves reading files, listing directories, running shell commands, or searching code.

The only time to prefer native tools is when lean-ctx explicitly can't cover it.
